class funzo.models.MDP(domain, reward, transition, discount=0.9)[source]¶

Markov Decision Process (MDP) model

For general MDPs, states and action can be continuous making it hard to efficiently represent them using standard data structures. In the case of discrete MDPs, it is straightforward to use array of comparable objects to represent the states.

In the continuous cases, we assume that only a sample of the state and action spaces will be used, and these can also be represented a simple hashable data structure (indexed by state or action ids).

Parameters:

domain : Domain object

The underlying domain (world) on which the MDP operates on

discount : float

MDP discount factor in the range [0, 1)

reward : RewardFunction object

Reward function for the MDP with all the relevant parameters

transition : MDPTransition object

Represents the transition function for the MDP. All transition relevant details such as stochasticity are handled therein.

Notes

This design deliberately leaves out the details of states and actions to be handled by the domain object which includes a reference to an MDP object. Additionally, transitions and reward which are in general functions are represented as separate callable objects with references to relevant data needed. This allows a unified interface for both discrete and continuous MDPs and further extensions

class funzo.models.MDPTransition(domain)[source]¶

A MDP transition function

\[T: \mathcal{S} \times \mathcal{A} \times \mathcal{S} \longmapsto \mathbb{R}\]

A generic way of representing MDP transition operation for both discrete and continuous spaces. A T function simply takes and action at a given state and executes it based on the transition properties (which could include stochasticity, etc)

class funzo.models.RewardFunction(domain, rmax=1.0)[source]¶

MDP reward function model

Rewards are as functions of state and action spaces of MDPs, i.e.

\[r: \mathcal{S} \times \mathcal{A} \longmapsto \mathbb{R}\]

Rewards for a state and action pair, \(r(s, a)\) are accessed via the __call__ method while appropriate reward function parameters are set in the constructor. In the MDP object, this function will be called via MDP.R function.

Parameters:

domain : Domain derivative object

Object reference to the domain/world of the MDP that the reward is to be used

rmax : float, optional (default: 1.0)

Upper bound on the reward function

class funzo.models.LinearRewardFunction(domain, weights, rmax=1.0)[source]¶

RewardFunction using linear function approximation

The reward funtion is define as,

\[r(s, a) = \sum_i w_i \phi_i(s, a)\]

where \(\phi_i(s, a)\) is a feature defined over state and action spaces of the underlying MDP. The weights are the parameters of the model and are usually assumed to sum to 1 to ensure that the reward remains bounded, a typical assumption used in most RL planners.
