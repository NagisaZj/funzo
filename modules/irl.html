Detailed descriptions¶

class funzo.irl.IRLSolver(mdp_planner=None)[source]¶

IRL algorithm interface

This interface assumes the most general formulation of the IRL problem. The task requires at minimum only expert demonstrations.

Notes

In case the underlying MDP is needed by an algorithm, a utility function for solving the MDP is included for convenience.

class funzo.irl.BIRL(planner, prior, inference='PW', beta=0.7, delta=0.2, burn_ratio=0.27, max_iter=100, random_state=None)[source]¶

Bayesian IRL algorithm

BIRL algorithm that seeks to find a reward function underlying a set of expert demonstrations by computing the posterior of the reward distribution \(p(r | \Xi)\).

These algorithms typically summarize the distribution by taking a single value such as the mean.

Parameters:

planner : a callable

A planner for MDP e.g. policy iteration as a callable

prior : RewardPrior or derivative object

Reward prior callable object

inference : str, optional (default=’PW’)

Inference procedure [‘PW’, ‘MPW’, ‘MAP’]

beta : float, optional (default=0.7)

Expert optimality parameter for the reward likelihood term in the product of exponential distributions

Attributes

_prior (RewardPrior or derivative object) Reference to the reward prior callable object
_beta (float, optional (default=0.9)) Expert optimality parameter for the reward likelihood term in the product of exponential distributions

class funzo.irl.RewardPrior(dim)[source]¶

Reward prior interface

The reward prior summarizes information about the reward distribution that is available before running the algorithm, i.e. all the relevant domain knowledge.

These distributions are multivariate, i.e. samples are vectors
